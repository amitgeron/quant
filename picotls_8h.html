Runs the handshake by dealing directly with handshake messages.

Callers MUST delay supplying input to this function until the epoch of the input becomes equal to the value returned by ptls_get_read_epoch().

Parameters
tlsthe TLS context
sendbufbuffer to which the output will be written
epoch_offsetsstart and end offset of the messages in each epoch. For example, when the server emits ServerHello between offset 0 and 38, the following handshake messages between offset 39 and 348, and a post-handshake message between 349 and 451, epoch_offsets will be {0,39,39,349,452} and the length of the sendbuf will be 452. This argument is an I/O argument. Applications can either reset sendbuf to empty and epoch_offsets and to all zero every time they invoke the function, or retain the values until the handshake completes so that data will be appended to sendbuf and epoch_offsets will be adjusted.
in_epochepoch of the input
inputinput bytes (must be NULL when starting the handshake on the client side)
inlenlength of the input
propertiesproperties specific to the running handshake
Returns
same as ptls_handshake

int ptls_handshake ( ptls_t *  tls,
ptls_buffer_t *  sendbuf,
const void *  input,
size_t *  inlen,
ptls_handshake_properties_t *  args 
)

proceeds with the handshake, optionally taking some input from peer.

The function returns zero in case the handshake completed successfully. PTLS_ERROR_IN_PROGRESS is returned in case the handshake is incomplete. Otherwise, an error value is returned. The contents of sendbuf should be sent to the client, regardless of whether if an error is returned. inlen is an argument used for both input and output. As an input, the arguments takes the size of the data available as input. Upon return the value is updated to the number of bytes consumed by the handshake. In case the returned value is PTLS_ERROR_IN_PROGRESS there is a guarantee that all the input are consumed (i.e. the value of inlen does not change).
